STRUKTURA JSON (FullDocumentResult):

{
  "process_id": "uuid",
  "document_pair_id": "uuid",
  "generated_at": "timestamp",
  "statistics": {
    "total_paragraphs": int,           // Wszystkie paragrafy w nowym dokumencie
    "unchanged_paragraphs": int,       // Niezmienione
    "modified_paragraphs": int,        // Zmodyfikowane
    "added_paragraphs": int,           // Nowe paragrafy
    "deleted_paragraphs": int,         // Usunięte paragrafy
    "total_changes": int,              // Suma: modified + added + deleted
    "tables_count": int,               // Liczba tabel
    "modified_cells": int              // Zmodyfikowane komórki w tabelach
  },
  "paragraphs": [                      // Lista wszystkich paragrafów
    {
      "index": int,                    // Pozycja w dokumencie (0-based)
      "text": str,                     // AKTUALNA treść (z nowego dokumentu)
      "type": "unchanged" | "modified" | "added" | "deleted",
      "old_text": str | null,          // Tylko dla "modified" i "deleted"
      "changes": [                     // Tylko dla "modified" - szczegółowe zmiany
        {
          "operation": "equal" | "delete" | "insert",
          "text": str                  // Fragment tekstu
        }
      ]
    }
  ],
  "tables": [                          // Lista tabel
    {
      "index": int,                    // Pozycja tabeli w dokumencie
      "rows": [[str]],                 // Wiersze i kolumny (nowy dokument)
      "changes": [                     // Lista zmian w komórkach
        {
          "table_index": int,
          "row_index": int,
          "col_index": int,
          "old_value": str,
          "new_value": str,
          "changes": [...]             // Szczegóły zmian w komórce
        }
      ]
    }
  ]
}

TYPY PARAGRAFÓW (pole "type"):
- "unchanged" → Identyczny w obu dokumentach, brak "old_text", brak "changes"
- "modified"  → Zmieniony, ZAWSZE ma "old_text" i "changes" z listą operacji
- "added"     → Nowy paragraf, brak "old_text", brak "changes"
- "deleted"   → Usunięty, ZAWSZE ma "old_text", brak "text" (lub text pusty)

OPERACJE w "changes" (pole "operation"):
- "equal"  → Fragment bez zmian (zachowany w obu wersjach)
- "delete" → Fragment USUNIĘTY (był w old_text, nie ma w text)
- "insert" → Fragment DODANY (nie było w old_text, jest w text)

OBSZARY ANALIZY RYZYKA (Bank w Polsce):

1. RYZYKO KREDYTOWE:
   - Zmiany w definicjach ekspozycji kredytowej
   - Limity, wskaźniki, współczynniki
   - Kryteria oceny zdolności kredytowej
   - Zasady klasyfikacji i rezerw
   - Zabezpieczenia kredytowe

2. RYZYKO OPERACYJNE:
   - Zmiany w procesach i procedurach
   - Podział obowiązków i odpowiedzialności
   - Kontrole wewnętrzne
   - Ciągłość działania (BCP/DRP)
   - Outsourcing i usługi IT

3. RYZYKO FINANSOWE:
   - Ryzyko rynkowe (FX, stopy procentowe)
   - Ryzyko płynności
   - Limity i ekspozycje
   - Wycena instrumentów finansowych

4. RYZYKO COMPLIANCE (Zgodność Regulacyjna):
   - Niezgodność z prawem polskim
   - Niezgodność z rekomendacjami KNF
   - Niezgodność z regulacjami UE
   - Ryzyka kar finansowych i sankcji

5. RYZYKO ESG:
   - Ryzyko środowiskowe (Environmental)
   - Ryzyko społeczne (Social)
   - Ład korporacyjny (Governance)
   - Raportowanie niefinansowe

REGULACJE DO WERYFIKACJI (Bank w Polsce):

✓ DORA (Digital Operational Resilience Act)
  - Odporność operacyjna ICT
  - Zarządzanie ryzykiem ICT
  - Testowanie odporności
  - Zarządzanie ryzykiem stron trzecich (TPP)
  - Raportowanie incydentów ICT
  - Wymiana informacji o zagrożeniach

✓ AML/CFT (Anti-Money Laundering / Counter Financing of Terrorism)
  - Przeciwdziałanie praniu pieniędzy
  - Przeciwdziałanie finansowaniu terroryzmu
  - Procedury należytej staranności (Due Diligence)
  - Raportowanie transakcji podejrzanych (STR)
  - Ustawa o przeciwdziałaniu praniu pieniędzy (polska)
  - Rekomendacje FATF

✓ KYC/CDD (Know Your Customer / Customer Due Diligence)
  - Weryfikacja tożsamości klienta
  - Należyta staranność wobec klienta
  - Enhanced Due Diligence (EDD) dla klientów wysokiego ryzyka
  - Beneficial Owner (rzeczywisty beneficjent)
  - PEP (Politically Exposed Persons)

✓ FATCA (Foreign Account Tax Compliance Act)
  - Identyfikacja klientów amerykańskich
  - Raportowanie do IRS (US Internal Revenue Service)
  - Umowa międzyrządowa USA-Polska (IGA)

✓ CRS (Common Reporting Standard - OECD)
  - Automatyczna wymiana informacji podatkowych
  - Identyfikacja rezydencji podatkowej klienta

✓ RODO/GDPR (Ochrona Danych Osobowych)
  - Przetwarzanie danych osobowych
  - Podstawy prawne przetwarzania
  - Prawa osób, których dane dotyczą
  - Bezpieczeństwo danych
  - UODO (Urząd Ochrony Danych Osobowych)

✓ MiFID II / MiFIR (Markets in Financial Instruments Directive)
  - Ochrona inwestorów
  - Przejrzystość rynku
  - Wymogi organizacyjne
  - Najlepsza realizacja (Best Execution)

✓ Bazylea III / IV (Capital Requirements)
  - CRD IV / CRR (Capital Requirements Directive/Regulation)
  - Współczynniki kapitałowe (CET1, Tier 1, Total Capital)
  - Współczynnik dźwigni (Leverage Ratio)
  - NSFR, LCR (stabilność finansowania, płynność)

✓ Rekomendacje KNF (Komisja Nadzoru Finansowego):
  - Rekomendacja D (zarządzanie ryzykiem kredytowym)
  - Rekomendacja H (kontrola wewnętrzna, zarządzanie ryzykami)
  - Rekomendacja I (zarządzanie ryzykiem płynności)
  - Rekomendacja P (zarządzanie ryzykiem modeli)
  - Rekomendacja U (zarządzanie ryzykiem braku zgodności)
  - Inne rekomendacje specyficzne dla sektora

✓ Ustawa Prawo Bankowe (Polska)
  - Wymogi kapitałowe
  - Tajemnica bankowa
  - Nadzór nad bankami

✓ PSD2 (Payment Services Directive 2)
  - Usługi płatnicze
  - Silne uwierzytelnianie (SCA)
  - Open Banking

TWOJE ZADANIA ANALITYCZNE:

1. Przeanalizuj "statistics" - oceń SKALĘ zmian (ile %, czy masowe czy punktowe)

2. Klasyfikuj zmiany według WPŁYWU (na podstawie treści z JSON):

   🔴 KRYTYCZNE (Immediate Action Required):
   - Zmiany w definicjach kluczowych pojęć
   - Zmiany w wartościach liczbowych (limity, wskaźniki, progi)
   - Zmiany w wymogach prawnych i regulacyjnych
   - Usunięcie wymaganych kontroli lub procedur
   - Zmiany w podpisach, uprawnieniach, zakresach odpowiedzialności
   - Zmiany wpływające na zgodność z regulacjami

   🟠 ISTOTNE (Important):
   - Zmiany w nazwach stanowisk, departamentów, komórek
   - Zmiany w terminach i deadline'ach
   - Zmiany w procedurach operacyjnych
   - Dodanie nowych wymagań
   - Zmiany w zakresach obowiązków

   🟡 MNIEJSZE (Minor):
   - Poprawki stylistyczne i językowe
   - Formatowanie
   - Zmiana kolejności bez zmiany treści
   - Doprecyzowania bez wpływu merytorycznego

3. Dla KAŻDEJ zmiany KRYTYCZNEJ i ISTOTNEJ:
   - Zidentyfikuj typ zmiany: modified/added/deleted
   - Podaj index paragrafu
   - Zacytuj dokładnie: old_text → text (lub tylko text dla added, tylko old_text dla deleted)
   - Oceń WPŁYW na bank
   - Wskaż OBSZAR RYZYKA (kredytowe/operacyjne/finansowe/compliance/ESG)
   - Wskaż REGULACJĘ (DORA/AML/KYC/FATCA/RODO/MiFID/Bazylea/KNF/inne)
   - Zaproponuj REKOMENDACJĘ (co zrobić)

4. Przeanalizuj "tables":
   - Dla każdej zmiany w komórce: old_value → new_value
   - Oceń wpływ zmiany (czy dotyczy wartości liczbowych, definicji, procedur)

5. Statystyki:
   - Policz zmiany KRYTYCZNE, ISTOTNE, MNIEJSZE
   - Policz zmiany według typu: modified/added/deleted
   - Policz zmiany według obszaru ryzyka
   - Policz zmiany według regulacji
